# Changelog - Framework Cortex

Histórico de versões e mudanças do Framework Cortex.

## Versão Atual

**Framework Cortex v1.2.0** - 2026-02-23

Para ver o changelog detalhado, consulte [CHANGELOG.md](CHANGELOG.md).

## Estrutura do Changelog

O changelog segue o formato [Keep a Changelog](https://keepachangelog.com/):

```
## [X.Y.Z] - YYYY-MM-DD

### 🚀 Novas Funcionalidades
- Descrição da nova funcionalidade

### ⚡ Melhorias de Performance
- Descrição da melhoria

### 🛡️ Melhorias de Segurança
- Descrição da melhoria

### 🔄 Mudanças de Comportamento
- Descrição da mudança

### 🐛 Correções
- Descrição da correção

### 📝 Notas Importantes
- Informações adicionais relevantes
```

## Categorias de Mudanças

| Categoria | Ícone | Descrição |
|-----------|--------|-----------|
| Novas Funcionalidades | 🚀 | Novos recursos adicionados |
| Melhorias de Performance | ⚡ | Otimizações de performance/token economy |
| Melhorias de Segurança | 🛡️ | Melhorias de segurança/robustez |
| Mudanças de Comportamento | 🔄 | Alterações no comportamento existente |
| Correções | 🐛 | Bugs corrigidos |
| Notas Importantes | 📝 | Informações adicionais |

## Semântica de Versionamento

**[MAJOR].[MINOR].[PATCH]**

- **MAJOR**: Mudanças incompatíveis ou grandes reestruturações
- **MINOR**: Novas funcionalidades (backward compatible)
- **PATCH**: Correções e pequenas melhorias

## Roadmap

Planejamento de versões futuras está em [CHANGELOG.md](CHANGELOG.md#roadmap).

## Como Contribuir

Ao fazer mudanças no Framework Cortex:

1. Atualizar o número da versão nos masters (cortex-boot.md, cortex-strategy.md, cortex-execution.md)
2. Adicionar entrada em CHANGELOG.md seguindo o formato acima
3. Documentar:
   - Motivo da mudança
   - Impacto em tokens/performance
   - Quebra de compatibilidade (se aplicável)
4. Atualizar a seção de Roadmap se necessário
